# 透明耗材校准与混色

[English](README.md) | **简体中文**

用于为透明耗材挑选打印配方（用哪种耗材、打多厚）以命中每块彩色玻璃的目标颜色的工具。
透明耗材没有标准的 CMYW 组合，切片软件的混色预览与真实背光打印相差很大，
因此我们对每种耗材做经验性校准。

校准 + 求解的核心（`analyze_calibration.py`、`solve_recipe.py`、`mixture.py`）
**仅依赖 numpy + PIL**（不用 scipy / OpenCV）——单应变换、斑点检测、诊断图都是手写实现的。
RAW 解码为可选（`rawpy`）；面板组装器 `svg_to_3mf.py` 用到 `shapely`（矢量化本就需要它）。

## 工作流程

### 1. 生成校准板 —— `make_calibration_pad.py`

一块连续的透明 **底板**（默认 0.4 毫米），其上构建方形单元格的网格，使整体为
**一个刚性部件**——无论你怎样把它放到屏幕上，单元格相对于标记的位置都保持不变。
每个单元格的总光程为 `底板 + 增量`（增量 0.1 → 2.0 毫米，步长 0.1 = 20 个单元格
→ 总厚 0.5 → 2.4 毫米）；我们用透过率拟合这个总厚度，所以底板只是每块板的一部分，
不会影响拟合。四个角默认是 **方形孔洞**（背光时透亮）——这样整板只用 **你的一种透明耗材、
无需换料**——在分析器中手动点选它们即可。（`--black-markers` 会恢复不透明的黑色角标 +
一个方向点，自动识别能找到它们，但需要换料。）**参考窗口是穿透板的真实孔洞**，
给出真正的裸屏采样。

```bash
python3 filament/make_calibration_pad.py          # -> filament/pad/{calibration_pad.3mf, layout.json, preview.png}
```

尺寸设计为落在手机屏幕 *内部*（默认 64×138 毫米有效区域 → 58×132 毫米的板）。
以 0.1 毫米层高打印（能整除底板和步长），使厚度精确。`layout.json` 以毫米记录每个
单元格 / 参考窗口 / 标记的位置以及每个单元格的总厚度——分析器会读取它。

### 2. 拍摄打印好的板

把板放在显示 **全屏纯色** 的手机/平板上（屏幕既是背光，也是可编程的 RGB 光源）。

**要拍几张？** `analyze` 会对每种耗材分类，并打印精确的 **拍摄要求**：
- **普通透明耗材** → **一张白屏即可。** 白光携带全部三个通道，每个都能干净拟合。
- **强吸收耗材**（把某通道吸得太狠，以致白光下发黑——例如深蓝把红色吃掉）→
  报告会说明 *还需拍哪些单色屏*（例如「WHITE + RED + GREEN」）。在全亮度的单色屏上，
  你可以把那一个通道过曝而不裁掉其它通道，把一个触底/噪声大的通道变成真实的拟合。
  即使在自己的屏下仍发黑的通道，说明该处确实不透光——其吸收系数以下限报告
  （无妨：它在任何混色中都读作 ~0）。

报告还会给出 **曝光要求**：裸屏（参考窗口）应读到满值的 **~85–95%**——亮但不过曝——
并把你的照片标为 OK / 太暗 / 太亮。

显示自带的标准三原色 `filament/screens/{white,red,green,blue}.png`
（纯 sRGB #FFFFFF/#FF0000/#00FF00/#0000FF；用
`python3 filament/make_screens.py --width W --height H` 按你的屏幕分辨率重新生成）。

实操要点：关闭 True Tone / 夜览 / 自动亮度，避免眩光，拍 RAW。
**每种屏幕颜色单独曝光**——见下文。

**尽量拍 RAW（DNG/ARW/CR2/…）**——它是线性的，跳过手机的色调曲线（否则会把吸收虚高
约 40–60%）。rawpy 会自动解码（`pip install rawpy`）。参考曝光：ISO 100、暗室、屏幕
固定在最大亮度、快门设到让 **该屏幕** 的裸窗口读到 ~85–95%（亮但不过曝）。
**每种颜色分别设快门，而不是四种共用一个** —— 透过率是 *同一张照片内* 的
单元格 ÷ 裸屏，绝对水平会抵消，因此每种颜色都应单独曝光；蓝屏比白屏暗得多，
用白屏调好的快门会让它欠曝（延长快门、保持低 ISO）。白平衡无关紧要——它在比值里抵消，
RAW 更让它无意义。`analyze` 会打印 **拍摄质量警告** 和逐屏曝光检查
（欠曝/过曝、噪声大的拟合）并给出重拍提示。透过单元格之间缝隙的裸屏正是用来
归一化掉曝光/亮度的。

### 3. 分析照片 —— `analyze_calibration.py`

```bash
python3 filament/analyze_calibration.py analyze \
    --layout filament/pad/layout.json --name "PolyTerra Teal" \
    --white white.jpg --red red.jpg --green green.jpg --blue blue.jpg \
    --out-dir filament/cal_polyterra_teal
```

每张照片会定位 4 个角——默认为背光透亮的角 **孔洞**（或在 GUI 里 / `--markers`
手动点选），或用 `--black-markers` 自动识别黑色角标——拟合从板毫米到图像像素的
单应变换，采样每个单元格和每个裸屏参考窗口，用穿过参考窗口拟合的平面去除
（屏幕亮度梯度 / 暗角）来归一化单元格，然后逐通道拟合

    ln T = b − a·t        a = 每毫米吸收，  T0 = exp(b) = 表面项

输出：`calibration.json`（头条 `primary_absorption_per_mm` = 红/绿/蓝屏的对角线），
外加 `detect_*.png` 叠加图和一张 `curves.png` 透过率曲线。

### 4. 求解打印配方 —— `solve_recipe.py`

给定各耗材的校准与一组目标颜色，找出在白光背光下最接近每个目标的
耗材 + 厚度的叠层（最小化 Lab Delta-E）。单一耗材够用时就用一种；否则求解器
从可用集中混合至多 `--max-filaments`（默认 3）种。

```bash
python3 filament/solve_recipe.py solve \
    --cal-dir filament/cals \
    --from-svg-dir sample1_fragments \
    --out-dir filament/recipes
```

直接从 SVG 生成器的 `color_NN_<hex>` 碎片文件名读取目标（或用
`--targets a1b2c3,ffcc00`）。写出 `recipes.json` 和一张 `swatches.png`
（目标色 vs 预测色）。报告的 Delta-E 对 **色域** 是诚实的：透明耗材只能做减法，
所以给定组合无法到达的高饱和目标会显示很高的 Delta-E——补一种能透过缺失原色的耗材。

模型：逐通道 `T_c = T0_c * exp(-sum_i a_ic * t_i)`；白光下的预测线性色 = 透过率。
叠层共用一个表面项 `T0`（熔合后的打印只有一对外界面）——将来测得的双层叠会细化交叉项。
单一耗材会精确复现其校准。

### 5. 纵览整个调色板 —— `solve_recipe.py map`

随着你校准更多耗材，`map` 给出调色板的一览：每种耗材的逐通道吸收、其强度类别 +
建议的混色上限、若干厚度下的背光色块，以及一张 **配对覆盖矩阵**
（哪些配对是直接 σ 校准的 `D`，哪些只是可泛化的 `g`）。

```bash
python3 filament/solve_recipe.py map --cal-dir filament/cals \
    --mixcal filament/mixcal/mixture_calibration.json \
    --mixcal filament/mixcal_gb/mixture_calibration.json   # -> filament_map.png
```

**极强吸收** 的耗材（某通道近乎不透光，例如蓝色在 R 通道 8.7/mm）会被标注一个建议的
最大混色比（`min(0.40, 3/max_a)`）：作为少数占比时它能参与，作为多数时会盖过其它颜色，
*且* 其泛化 σ 预测会变差（红–蓝：蓝≤40% ΔE 7.5，蓝>40% ΔE 13.4）。
分层求解器会对混色自动执行这个上限（纯色块不受限）。

**只用可打印的比例。** 切片器交错整数层的子层，无法构建任意的 11% 或 57%。
求解器在一个可构建比例的查找表里搜索——十分之几加简单分数
（`0,10,20,25,30,33,40,50,60,67,70,75,80,90,100`%）——并把每个混色报告为其整数
**子层计数**（例如 `blue 20% / red 80% [1:4]`，三分之二记为 `2:1`）。强吸收耗材的上限
会进一步收窄（蓝 ≤35% → 只剩 10/20/25/30/33%），于是整个配方空间是一个小而可打印的集合。

### 导出可打印颜色调色板——用真正能打印出来的颜色作画

在 GUI 的 **Colour LUT** 标签页里，先勾选你要装载的耗材，再点 **导出可打印颜色**：
它会为该组合构建 LUT，并把每一种可打印颜色写成可导入绘图软件的调色板，
这样你就只用面板能还原的颜色来设计（不会出现超色域的意外）：

- `palette.gpl` —— GIMP / Krita / Inkscape / Aseprite
- `palette.aco` —— Adobe Photoshop（v1 + 带名称的 v2 块）
- `palette.png` —— 带十六进制标注的色卡

颜色会按预测的十六进制去重；可选的 **最多颜色** 会把集合缩减为最有区分度的 N 种
可打印色块（Lab k-means）。

## 分层混色（实验性，Bambu Studio Color Mixing）

叠整层色块之外的另一种方式：在一个实心部件内，Bambu Studio 按某个比例交错地打印
2–3 种耗材的薄子层，背光下人眼会将它们混合。物理基线与整层相同
（对混色比例做 Beer–Lambert），再加上一个单独校准的界面散射项。

**校准一对耗材** —— `make_mixture_pad.py` 生成一条 **连续的 A→B 渐变条**：
一行相邻的单元格，每个是一个实心部件、**标注了它的混色比**（子层切分由 Bambu 完成）。
它读起来是一条平滑的条带——便于肉眼判断，也便于 CV 分割。输出的 `mixture_pad.3mf`
可作为真正的 Bambu 工程打开，所有比例都已预设（无需拖拽色三角）：

```bash
python3 filament/make_mixture_pad.py                       # 0→100 %B，步长 10%
python3 filament/make_mixture_pad.py --start 0 --end 50 --ramp-step 5   # 放大前半段
# -> filament/mixpad/{mixture_pad.3mf, layout.json, preview.png}
```

`--start/--end` 选择子范围，`--ramp-step` 是每格的 %B（格数 = 范围/步长 + 1）；
`--cell-w/--cell-h/--depth-mm` 设定条带尺寸。**没有打印标记**——这条渐变是一个干净的
矩形，你手动点选它的 4 个物理角点；分析器从纯色端（等于熨烫过的单色校准）固定曝光，
并自动判断哪一端是 A、哪一端是 B。

它以自带的 `filament/templates/bambu_p2s.3mf`（一个净化过的 Bambu Lab P2S /
PETG-Transparent 骨架）为模板。若为其它机型，用 `--bambu-template <export>.3mf`
传入你自己的 Bambu `.3mf` 导出；`--plain` 则写出一个普通 core-3MF
（Bambu 会标记为「非 Bambu Lab」）。单色的 `make_calibration_pad.py` 接受相同参数。

**混色模型** —— `mixture.py`。需要每种耗材的整层校准作为基线（`ln T = b − a·t`，
在纯色端精确）。它从每对渐变的不对称性拟合每种耗材的散射 `σ_X`，跨配对联合求解，
使共享耗材保持一致。然后 `T_c = T0·exp(−[base + scatter])`（其中
`scatter = T·Σ_i σ_i·f_i²(1−f_i)`）能预测 **任意** 配对（包括从未打印过的——
A–C 由 A–B + B–C 得到）以及用同一批配对校准 σ 预测 **三耗材** 混色。
当某配对自身实测的渐变与泛化预测不一致时，以实测为准（后验覆盖）。

```bash
python3 filament/mixture.py fit --layout filament/mixpad/layout.json --white mix.dng \
    --a red=filament/cals/red/calibration.json --b green=filament/cals/green/calibration.json \
    --out-dir filament/mixcal        # -> mixture_calibration.json（每种耗材的 σ）
python3 filament/mixture.py selftest # 恢复 σ；预测未见过的 A-C + 三色混合
```

**合并多条渐变**（`--pads-spec`，或直接在 GUI 里添加渐变条）：打印一条粗略的
0–100 %@20% 加一条精细的 0–50 %@10%，来细调某个饱和区域——所有点进入 **同一次联合拟合**，
密集覆盖处的 σ 会更精确。部分范围的渐变从其唯一的纯色端做曝光参考
（其非纯色端保留真实散射）。只有渐变 **中段** 驱动 σ；若某纯色端出现混色板的打印纹理伪影，
会被自动锚定到熨烫过的单色校准值，因此你无需熨烫并重打整条渐变。

**求解 / 预测分层配方** —— 拟合出的 σ 直接喂给
`solve_recipe.py --mode sub-layer`（每对已校准配对传一个 `--mixcal`；它们会合并，
所以共享耗材的 σ 能覆盖未见过的配对——A-C 由 A-B + B-C 得到）：

```bash
# 正向：预测某个具体混色的背光色
python3 filament/solve_recipe.py predict --cal-dir filament/cals \
    --mixcal filament/mixcal/mixture_calibration.json \
    --mix red=0.5,green=0.5 --thickness 1.0

# 求解：为每个目标色找最佳的（耗材对 + 比例 + 厚度）
python3 filament/solve_recipe.py solve --mode sub-layer --cal-dir filament/cals \
    --mixcal filament/mixcal/mixture_calibration.json \
    --from-svg-dir sample1_fragments --layer 0.2 --out-dir filament/recipes_sub
```

两者对任何没有混色校准的耗材都会回退到 σ=0（纯吸收）并给出警告——该基线在渐变中段
会偏离约 dE 15，所以请先校准该配对。

**真实打印的发现。** 红+绿都在打印的 0.2 毫米层高下校准后，一条打印的 11 级分层渐变
验证了模型：基线（仅单耗材吸收）平均 ΔE **11.7**（中段可达约 20），σ 模型平均 ΔE
**4.1**——已到打印可重复性的下限（纯色端点本身就有 ΔE ~6 的批次间波动）。
因此单耗材校准对分层混色 **并不** 够；你需要配对渐变的 σ，但它能按耗材泛化。
吸收还与层高相关——绿色的红光吸收从 0.1 到 0.2 毫米上升了 +46%——所以要在生产层高下校准。

**`bambu_mix3mf.py`** —— 可复用的写入器，把各部件（每个标注了基底耗材或混色比）
组装成 Bambu 混色 3MF：将不同的混色去重为虚拟耗材槽位，把每个逐耗材的配置数组
扩展到新的槽位数（处理双喷头 ×2 变体数组），并保留模板的 Bambu 标记，
使其作为真正的工程加载。它还能嵌入额外的包内文件，并把某个部件标记为可编辑的
**`BambuStudioShape`** SVG。

### 组装面板 —— `svg_to_3mf.py`

最后一段把所有环节串起来：读取 SVG 生成器的 `color_NN_<hex>` 玻璃块，
把每块对齐到 LUT 里最接近的 **可打印** 配方，挤出为一块平板，
再通过 `bambu_mix3mf.py` 写出 **一个 Bambu 混色 3MF**。

```bash
python3 filament/svg_to_3mf.py --frag-dir myimage_fragments \
    --cal-root filament/calibration --out panel.3mf \
    --max-delta-2 10 --max-delta 20 \       # 单色 ≤10 ΔE，否则双色 ≤20 ΔE，否则三色
    --leading myimage_leading.svg            # 把铅条嵌在上方
```

- 按耗材数量分级：单色在 `--max-delta-2` 阈值内则用 **单色**，否则在 `--max-delta`
  阈值内用 **双色混合**，再否则用 **三色混合**。`--num-colors` 精简调色板；
  `--filaments` 限定为你已装载的 AMS 槽位。
- `--leading` 把铅条 SVG 作为凸起、可编辑的部件嵌在面板之上（见根 README）。
  当某配对没有混色校准时，`--no-sigma` 使用直接近似（σ=0）——无需打印任何渐变即可工作，
  略有偏差。
- 可由 `glass_gui.py` 驱动（图像 → 玻璃块 → 3MF，带实时色域预览）。

### 自检（无需打印机）

分析器能从已知真值的耗材渲染合成的背光板照片（透视畸变 + 亮度梯度 + 噪声），
并检查它是否恢复出系数——这正是本流水线在任何真实打印之前得到验证的方式：

```bash
python3 filament/analyze_calibration.py selftest --out-dir /tmp/cal
# -> SELF-TEST PASSED，恢复的吸收与真值相差 <0.01/mm
python3 filament/solve_recipe.py selftest
# -> SELF-TEST PASSED，植入的配方恢复到 dE<=2
python3 filament/mixture.py selftest
# -> SELF-TEST PASSED，未见过的 A-C 配对 + 三耗材混色预测到 dE<1.5
```

## 路线图

- [x] 校准板生成器（`make_calibration_pad.py`）—— 一个刚性底板部件
- [x] 照片分析器（`analyze_calibration.py`），已用合成数据验证
- [x] 整层求解器（`solve_recipe.py`）：调色板 → 每块配方
      （耗材 + 厚度 + 预测色 + ΔE），叠层 ≤3 种耗材
- [x] 在真实打印并拍摄的板上验证（红+绿，0.2 毫米）
- [x] 分层混色校准（`mixture.py fit`）+ 求解器
      （`solve_recipe.py --mode sub-layer`），真实打印验证（ΔE 11.7→4.1）
- [x] 第三种耗材（蓝）：在未见过的红–蓝配对上测试 σ 泛化
      （基线 ΔE 18.8 → 泛化 10.7 → 直接/后验 5.2）
- [x] 强吸收耗材检测 + 混色比上限（谱侧退让、只用淡色稀释剂）；可打印比例查找表
- [x] 颜色 LUT + 色域（`solve_recipe.py lut`）；调色板图（`map`）；
      调色板推荐器（为一张图排序耗材子集）
- [x] 连续 **渐变条** 混色板 + **多条联合** σ 拟合（合并粗略 + 精细范围）；
      端点锚定到熨烫过的单色校准
- [x] **组装面板**（`svg_to_3mf.py`）：SVG 玻璃块 → LUT → 每块配方 →
      一个 Bambu 混色 3MF，带分级的 1/2/3 色阈值、直接近似（σ=0）回退，
      以及作为可编辑 `BambuStudioShape` 嵌入的铅条

## 目录结构与工作流

校准结果集中在一个根目录下，使各工具无需路径即可衔接：

```
filament/calibration/
  <name>/calibration.json                 # 每种耗材（analyze_calibration.py）
  mix/<A>+<B>/mixture_calibration.json     # 每对耗材   （mixture.py fit）
  color_lut.json, gamut.png, filament_map.png   # 生成物 （solve_recipe.py lut/map）
```

```bash
# 1. 校准每种耗材（一张白屏；自动存到 calibration/<name>/）
python3 filament/analyze_calibration.py analyze --layout filament/pad/layout.json \
    --name red --white photo/red-w.dng --layer-mm 0.2
# 2. 校准一组能张成整体的配对（自动存到 calibration/mix/<A>+<B>/）
python3 filament/mixture.py fit --layout filament/mixpad/layout.json --white photo/rg.dng \
    --a red=filament/calibration/red/calibration.json --b green=filament/calibration/green/calibration.json
# 3. 构建 LUT + 色域（读取整个校准根目录，无需路径）
python3 filament/solve_recipe.py lut --match ff8800,3366cc
```
